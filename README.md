# rogerogert.github.io
[![Build Status](https://travis-ci.org/rogerogert/rogerogert.github.io.svg?branch=release)](https://travis-ci.org/rogerogert/rogerogert.github.io)

Site pessoal desenvolvido com Jekyll.

O repositório está organizado da seguinte maneira:

* **master**
    * Contém apenas conteúdo gerado pelo build do projeto jekyll disponível na branch release.
* **release**
    * Branch utilizada pelo Travis para realizar o build do projeto e o deploy do site na branch master.
* **develop**
    * Branch que contém todas as modificações antes do merge com a branch release.
    
    
O projeto utiliza o serviço Travis-CI para realizar o build e o deploy automatizado do site.

O artigo [Supercharge GitHub Pages with Jekyll and Travis CI](https://medium.com/@mcred/supercharge-github-pages-with-jekyll-and-travis-ci-699bc0bde075) detalha melhor esse fluxo de deploy utilizando o Travis-CI.
